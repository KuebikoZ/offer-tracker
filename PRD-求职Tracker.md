## 1. 产品概述

### 1.1 产品定位

**产品名称**：求职Tracker（暂定）

**一句话描述**：帮助大学生统一管理求职投递记录、可视化追踪求职进度，并通过智能化分析发现求职盲区的工具。

### 1.2 目标用户

| 属性 | 描述 |
|------|------|
| 身份 | 在校大学生（大一至研二） |
| 场景 | 求职过程中（找实习/秋招/春招/日常招聘） |
| 求职方向 | 不限（互联网、金融、国企、外企、快消等均可） |
| 并发量 | 平均同时关注 30-50 家公司的投递记录 |

### 1.3 核心痛点

1. **信息分散**：投递记录散落在各招聘网站、邮箱、微信聊天中，难以统一管理
2. **进度黑盒**：提交投递后，不知道公司是否已读、是否进入下一阶段，需要反复登录各平台查看
3. **盲目投递**：不清楚自己的投递分布是否合理，是否有"一直投某行业但没回音"的情况
4. **状态遗忘**：一个秋招可能投递50+公司，时间久了忘记某些公司的进度

### 1.4 解决方案

| 痛点 | 解决方案 |
|------|----------|
| 信息分散 | 截图识别 + 手动录入，统一存储 |
| 进度黑盒 | 接入公司官网，自动抓取投递状态 |
| 盲目投递 | 多维度可视化分析（行业/公司/岗位分布） |
| 状态遗忘 | 投递列表 + 进度时间线 |

**状态说明**：

|------|------|

| 已拒绝 | 用户主动拒绝该公司 |

### 2.3 核心功能详细说明

#### 2.3.1 截图识别添加

**流程**：
1. 用户上传招聘网站的投递确认截图（支持任意图片格式）
2. 系统通过 OCR 识别关键信息：
   - 公司名称
   - 岗位名称
   - 投递时间
   - 招聘平台
3. 识别结果展示给用户，用户可修改/补充
4. 用户确认后保存到投递列表

**技术要点**：
- OCR 识别：接入百度/腾讯/阿里云 OCR API
- 支持图片类型：截图、拍照、PDF转图片
- 识别字段：尽可能自动识别，不可识别字段留空让用户填写

#### 2.3.2 进度追踪方案（双轨制）

**目标**：解决"不知道公司进度"的核心痛点

**轨道A：自动化检测（被动）**

| 方式 | 原理 | 实现难度 |
|------|------|----------|
| 公开页面抓取 | 定期访问公司招聘页/候选人追踪页，检测岗位是否仍在线 | 低 |
| 邮件追踪 | 监控用户绑定的邮箱，识别面试邀请/offer等进度通知邮件 | 中 |

- 公开页面：可检测「岗位已下线」= 投递流程可能结束
- 邮件追踪：可识别面试通知、测评邀请、offer发放等关键节点

**轨道B：用户手动更新（主动）**

- 投递列表支持一键「更新状态」按钮
- 状态变更有时间戳记录
- 提供快捷操作：「已投递」「进入一面」「收到offer」等

**为什么双轨制？**
- 自动化无法覆盖所有场景（很多公司无公开信息）
- 手动更新作为兜底，确保状态准确性
- 用户主动更新也能增强参与感

**技术要点**：
- 邮件追踪：接入 Gmail/QQ邮箱/163邮箱 API（需用户授权）
- 定时任务：APScheduler，每天固定时间执行
- 页面解析：BeautifulSoup，过滤动态渲染页面

#### 2.3.5 多维度可视化

**行业分布分析**：
- 饼图/环形图展示投递行业占比
- 支持按行业筛选投递列表
- 提示：投递过少或过多的行业

**公司分布分析**：
- 词云图或柱状图展示投递公司TOP榜单
- 区分「已投」「流程中」「已结束」状态

**岗位类型分析**：
- 按岗位类别聚合（技术/产品/运营/市场/职能等）
- 查看各类型的投递数量和反馈率

**阶段漏斗分析**：
- 展示每个阶段的投递数量和转化率
- 例：已投递100人 → 一面30人 → 二面10人 → 签约2人
- 识别哪个阶段流失最严重

**趋势分析**：
- 折线图展示每周/每月投递数量
- 识别投递高峰期（秋招9月、春招3月）

## 3. 非功能需求

### 3.1 性能需求

| 指标 | 要求 |
|------|------|
| 页面加载时间 | < 2秒 |
| 截图识别时间 | < 5秒 |
| 官网同步延迟 | < 24小时 |
| 支持数据量 | 单用户 500+ 条投递记录 |

### 3.2 安全需求

- 用户数据加密存储
- 敏感信息（如简历）脱敏处理
- 防爬虫机制（针对官网同步功能）

### 3.3 兼容性需求

- 浏览器支持：Chrome、Firefox、Safari、Edge 最新两个版本
- 响应式设计：支持 PC 端（主）、平板（次）、手机（基础）

### 3.4 数据存储

| 数据类型 | 存储方案 |
|----------|----------|
| 用户账号信息 | 云端数据库（MySQL/PostgreSQL） |
| 投递记录 | 云端数据库 |
| 上传的图片 | 对象存储（OSS/S3） |
| 可视化图表 | 前端渲染 |

## 4. 竞品分析

### 4.1 现有解决方案

| 产品 | 优点 | 缺点 |
|------|------|------|
| Excel/Notion | 灵活、可定制 | 需要手动维护、分散 |
| 实习僧/牛客 | 自带投递追踪 | 只限平台内、功能单一 |
| 求职日历APP | 有DDL提醒 | 无可视化、分析能力弱 |

### 4.2 我们的差异化

1. **截图即录入**：3秒完成一个投递记录
2. **官网自动同步**：无需手动登录各网站查看进度
3. **求职健康度诊断**：告诉你投递分布是否合理、哪里是盲区
